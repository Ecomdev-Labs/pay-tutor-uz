# API и вебхуки

Основные HTTP-эндпоинты (файловая организация):
- `payment_webhook.php` — POST, приём уведомлений о платеже от FreedomPay.
  - Ожидаемый payload: { "transaction_id", "status", "amount", "signature" }
  - Ответ: HTTP 200 при успешной обработке.
- `mock_freedompay.php` — тестовый эндпоинт для симуляции уведомлений.

Взаимодействие с Telegram API:
- `TelegramBot.php` использует методы Telegram для отправки сообщений, клавиатур и обработки callback'ов.
- Рекомендуется использовать вебхуки для Telegram или polling в зависимости от окружения.

Требования к валидации:
- Подписывайте и проверяйте payload вебхуков (проверка `signature`).
- Проверяйте сумму и идентификатор транзакции перед изменением статуса в БД.

Примеры ответов и ошибок описать в отдельном разделе по мере детализации.
