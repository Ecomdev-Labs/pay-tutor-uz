# Архитектура

Компоненты:
- Веб-приложение на PHP — основная логика и роутинг.
- Telegram-бот — реализован в `TelegramBot.php`, отвечает за обработку команд и состояний пользователей.
- Платёжный модуль — `FreedomPay.php` + `mock_freedompay.php` для тестирования.
- Вебхуки — `payment_webhook.php` принимает уведомления о статусах платежей.
- СУБД — взаимодействие через `database.php`.

Схема взаимодействия:
1. Пользователь инициирует оплату через бота.
2. Бот создаёт платёж через `FreedomPay.php` и хранит транзакцию в БД.
3. Провайдер отправляет уведомление на `payment_webhook.php`.
4. Сервис обновляет статус в БД и отправляет пользователю уведомление в Telegram.

Нефункциональные требования:
- Логирование ключевых событий (ошибки, платежи).
- Обработка повторных уведомлений (идемпотентность).
- Безопасность: проверка подписи вебхуков, хранение секретов вне репозитория.
