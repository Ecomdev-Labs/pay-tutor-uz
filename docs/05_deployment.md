# Развёртывание и окружение

Локальная разработка:
- OSPanel / OpenServer (Windows) — текущая среда разработчика.
- Настроить виртуальный хост, PHP версии 7.4+ или 8.x, расширение PDO.

Продакшн (рекомендации):
- Linux (Ubuntu/Debian) + Nginx/Apache, PHP-FPM.
- Настроить HTTPS (Let's Encrypt).
- Хранить секреты в переменных окружения (`.env`) и не в репозитории.
- Настроить логирование и ротацию логов.

Процедура деплоя:
1. Клонировать репозиторий на сервер.
2. Установить зависимости (если потребуется Composer).
3. Настроить конфиги и переменные окружения.
4. Применить миграции для БД.
5. Настроить systemd unit или crontab для задач (если необходим polling для бота).

Мониторинг и бэкап:
- Настроить мониторинг доступности вебхуков и очередей.
- Регулярный бэкап БД и проверка целостности.

**Production domain**
- Боевой домен кейс-сайта: **https://pay-tutor.ecomdev.uz** — зафиксирован в [DOMAIN.md](DOMAIN.md), `Config::PRODUCTION_SITE_ORIGIN`, `public/site_static/site.json`, `SITE_URL` в `.env`.
- Demo API (бот, webhook): **https://demo-api.pay-tutor.ecomdev.uz** — отдельный хост, не подменяет кейс-сайт.

**Cloudflare / Workers**
- Примечание: попытка деплоя на Cloudflare Workers в текущем аккаунте не прошла успешно — Worker URL отображается как "Inactive" и в панели замечено "Latest build failed" (см. скриншот). Не переключать трафик на этот хост до устранения проблемы.
- Рекомендуемые шаги для диагностики и исправления:
	1. Просмотреть логи сборки (Build logs) в разделе Deployments / Workers, найти ошибку компиляции/биндингов.
	2. Проверить настройки Bindings / Secrets (например, переменные окружения, привязки KV, R2 или других сервисов), которые требуются воркеру.
	3. Убедиться, что в настройках домена прописаны корректные маршруты (Routes) и DNS указаны на Cloudflare при необходимости.
	4. Попробовать ручной ребилд и деплой (Edit code → Deploy) и повторно проверить статус.
	5. Если ошибка не ясна — экспортировать логи и обратиться в поддержку Cloudflare или вернуть деплой на классический VPS/сервер как запасной вариант.

В документ TZ добавить пометку о боевом домене и ограничение по использованию Cloudflare до решения проблемы деплоя.
