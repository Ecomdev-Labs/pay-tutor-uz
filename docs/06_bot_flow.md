# Сценарии и логика бота

Основные сценарии пользователей:
- Регистрация / привязка Telegram-аккаунта.
- Запрос баланса / информации.
- Инициация платежа: пользователь выбирает сумму → бот создаёт платеж → отправляет ссылку/вызов оплаты.
- Ожидание подтверждения платежа и уведомление пользователя о результате.

Состояния диалогов:
- `idle` — ожидает команды.
- `pay_flow:await_amount` — ожидает ввод суммы.
- `pay_flow:await_confirmation` — ожидает подтверждение оплаты.

Обработка ошибок и краевых случаев:
- Таймауты платежей — помечать как `expired` и уведомлять пользователя.
- Повторные уведомления от провайдера — обрабатывать идемпотентно.
- Отказ в оплате — логировать причину и уведомлять пользователя с инструкцией.

Требования к UX:
- Чёткие сообщения при ошибках.
- Подтверждения действий перед списанием.
- Кнопки быстрых ответов и инлайн-клавиатуры для удобства.
