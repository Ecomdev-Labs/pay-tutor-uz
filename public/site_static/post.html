Telegram bot для онлайн‑школы — развёрнутый кейс

Опубликовано: 2026-05-28 · Кейс automation-услуги Ecomdev

Problem — задача клиента

Онлайн‑школам, репетиторам и авторам закрытых Telegram‑сообществ нужна автоматизация полного цикла продажи:

  • подбор формата занятий (индивидуальные, парные, групповые);
  • расчёт стоимости в зависимости от количества уроков;
  • приём оплаты через локальный платёжный шлюз (FreedomPay / PayBox);
  • автоматическая выдача цифрового продукта после подтверждения оплаты.

Без автоматизации каждый заказ требует ручного участия менеджера — это замедляет продажи и создаёт ошибки при выдаче доступа.

Solution — техническое решение

Реализован Telegram‑бот на PHP с хранением состояния диалога в SQLite. Бот использует inline‑клавиатуру и state machine: например, после выбора тарифа переходит в состояние waiting_for_lessons_count и принимает только корректный числовой ввод.

Заказы сохраняются в таблице orders со статусом pending. Платёжная ссылка генерируется классом FreedomPay с MD5‑подписью. Webhook‑обработчик принимает callback, атомарно обновляет статус на paid и инициирует fulfillment.

Воронка продаж

Команда /start сбрасывает состояние и показывает главное меню. Пользователь выбирает «Купить уроки» или «Доступ в закрытый канал». Для уроков бот запрашивает количество, умножает на цену тарифа и формирует заказ. Для канала — фиксированная сумма 300 000 UZS.

После оплаты webhook различает product_type: для lessons отправляется номер заказа и контакт преподавателя; для channel — одноразовая invite‑ссылка через createChatInviteLink.

Интеграции

  • Telegram Bot API — sendMessage, answerCallbackQuery, createChatInviteLink.
  • FreedomPay — init_payment, pg_sig, Result URL webhook.
  • SQLite / PDO — users, orders; миграция на MySQL через DATABASE_DSN.
  • Mock‑стенд — симулятор оплаты для demo без реальных секретов.

Модель публикации

Проект позиционируется как кейс кастомной automation-услуги на Ecomdev, а не как открытый self‑serve продукт. Публично доступны: описание воронки, mock‑скриншоты интерфейса, demo‑стенд. Webhook, токены и реальная касса не индексируются.
