Коротко о проекте

PayTutor — демонстрационный кейс кастомной automation-услуги для онлайн‑школ, репетиторов и авторов закрытых сообществ. Бот ведёт клиента по воронке, рассчитывает стоимость, принимает оплату через платёжный шлюз и автоматически выдаёт цифровой продукт: номер заказа, инструкции или одноразовую ссылку в закрытый канал.

Технические webhook и платёжные endpoint не индексируются и не публикуются как открытый продукт — публично доступен только этот кейс и безопасный mock‑демо.

Problem — задача

Онлайн‑школам и репетиторам нужно продавать уроки и доступ в Telegram без ручной работы на каждом шаге:

  • менеджер вручную принимает заявки и считает стоимость;
  • оплата происходит вне автоматизированного сценария;
  • после оплаты доступ выдаётся вручную — риск ошибок и задержек;
  • нет единой воронки «выбор → оплата → выдача».

Solution — решение

Telegram‑бот с state machine в базе данных, интеграцией FreedomPay и webhook‑обработчиком:

  • контекстный диалог с inline‑кнопками и запоминанием состояния;
  • динамический расчёт суммы по тарифу и количеству уроков;
  • безопасная оплата через платёжный шлюз с MD5‑подписью;
  • автовыдача: invite‑ссылка в канал или инструкции преподавателю;
  • мгновенное уведомление администратору о каждой продаже.

Схема воронки и автоматизации

Пользователь проходит путь внутри Telegram; backend обрабатывает заказ и webhook от платёжной системы.

  1. 1
    /start → главное меню

    Купить уроки, доступ в канал, юридическая информация, поддержка.

  2. 2
    Выбор продукта

    Индивидуальные / парные / групповые уроки или подписка на закрытый канал.

  3. 3
    Расчёт и заказ

    Бот запрашивает количество уроков, умножает на тариф, создаёт заказ в БД (pending).

  4. 4
    Оплата

    Клиент переходит по ссылке FreedomPay; провайдер отправляет callback на webhook.

  5. 5
    Fulfillment — выдача

    Уроки → номер заказа и контакт преподавателя. Канал → одноразовая invite‑ссылка через Telegram API.

Сценарий оплаты и выдачи цифрового продукта

Уроки

Клиент выбирает формат → вводит число уроков → оплачивает сумму → получает номер заказа и инструкцию связаться с преподавателем.

Закрытый канал

Фиксированная цена → оплата → бот вызывает createChatInviteLink с member_limit: 1 и отправляет персональную ссылку.

Администратор

После успешного webhook заказ помечается paid (идемпотентно); владелец получает сводку: сумма, тип продукта, chat_id клиента.

Интерфейс бота (безопасные mock‑скриншоты)

Демонстрационные макеты интерфейса без реальных данных, токенов и webhook URL.

Стек и интеграции

Telegram Bot API FreedomPay / PayBox PHP 7.4+ SQLite / PDO Webhook cURL MD5 signature State machine
  • Telegram Bot API — сообщения, inline‑клавиатуры, invite‑ссылки, уведомления админу.
  • FreedomPay — генерация платёжной ссылки, приём callback, проверка подписи (в production).
  • SQLite + PDO — пользователи, состояния диалога, заказы; поддержка MySQL через DATABASE_DSN.
  • Mock‑стенд — локальный симулятор оплаты для демо без реальных карт и секретов.
